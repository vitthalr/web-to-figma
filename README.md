# Web to Figma — Chrome Extension

Capture **any web page** (including authenticated/login-protected pages) and send it to Figma as **editable frames** — not screenshots.

Works on sites with strict Content Security Policy (CSP) and Trusted Types that block standard script injection.

Built on top of the [Figma MCP Server](https://developers.figma.com/docs/figma-mcp-server/) — the official Model Context Protocol interface for Figma. See the [GitHub Copilot to Figma](https://www.figma.com/release-notes/?title=github-copilot-to-figma) release notes for more context.

## Why This Exists

Figma's MCP server can already capture web pages into editable Figma frames — but only if it can inject a capture script into the page. Many **authenticated pages** enforce strict Content Security Policy (CSP) and Trusted Types, which block all script injection — even from browser extensions using standard methods.

This extension solves that by using the **Chrome Debugger API (CDP)**, which executes JavaScript at the browser engine level, bypassing all CSP and Trusted Types restrictions. It's the same mechanism Chrome DevTools uses — and it works on every page.

## How It Works

The extension combines two technologies:

1. **[Figma MCP Server](https://mcp.figma.com)** — Figma's official MCP server that generates capture IDs and converts captured DOM into editable Figma design frames
2. **Chrome Debugger API (CDP)** — Chrome DevTools Protocol to inject Figma's capture script into any page, bypassing CSP and Trusted Types restrictions

This means you can capture **authenticated pages** (internal tools, enterprise apps, admin dashboards) that normally block all script injection — and get **editable Figma frames** with real text, buttons, auto-layout, and proper styling.

## Setup

1. Open Chrome → go to `chrome://extensions/`
2. Enable **Developer mode** (toggle in top-right)
3. Click **"Load unpacked"**
4. Select this folder

## Usage

1. Set up the [Figma MCP server](https://developers.figma.com/docs/figma-mcp-server/remote-server-installation/#vs-code) in your IDE (VS Code, Cursor, etc.)
2. Ask your AI assistant (Copilot, Claude, etc.): *"generate a capture ID"*
3. Navigate to any page in Chrome
4. Click the extension icon → paste the capture ID
5. Click **"Capture to Figma"**
6. Accept the Chrome debugger prompt
7. Tell your assistant to check the status — it will give you the Figma link

## Permissions Explained

| Permission | Why It's Needed |
|---|---|
| `debugger` | Injects the capture script via Chrome DevTools Protocol, bypassing CSP/Trusted Types |
| `activeTab` | Accesses the current tab to capture its content |
| `scripting` | Runs scripts in the page context |
| `<all_urls>` | Needed to capture any website |

## Security

- **No tokens or credentials stored** — capture IDs are generated by your AI assistant via the Figma MCP server
- **No data collection** — the extension doesn't phone home or track anything
- **All code is readable** — nothing minified or obfuscated
- The `debugger` permission shows a yellow bar ("Extension is debugging this browser") while capturing — this is normal and goes away when capture is complete

## Requirements

- Google Chrome (or Chromium-based browser)
- An AI assistant with the [Figma MCP server](https://developers.figma.com/docs/figma-mcp-server/) configured
